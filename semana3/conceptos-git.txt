•Repositorio: un repositorio es el espacio central donde vive todo el código de un proyecto, junto con todo el historial de cambios que se han hecho. Es como una base de datos que Git maneja, permitiéndome trabajar en diferentes versiones del software sin perder nada. Puedo tener copias locales en mi máquina o alojarlas en servicios como GitHub para colaborar.
•Commit: Un commit lo veo como una "cápsula del tiempo". Cuando hago un commit, guardo de forma permanente el estado actual de los archivos que estoy rastreando, junto con una descripción de qué cambiaron. Es mi manera de crear puntos de control en el desarrollo: si arruino algo más adelante, puedo regresar a cualquier commit anterior como si nada hubiera pasado.
•Push: es el comando que uso para sincronizar mi trabajo local con el remoto. Una vez que tengo uno o varios commits hechos en mi computadora, al hacer push los envío al repositorio en la nube o en el servidor. Esto no solo respalda mi código, sino que hace que mis cambios estén disponibles para mis compañeros. Sin el push, todo mi trabajo quedaría atrapado únicamente en mi disco duro.
